# /배포 명령어

자동 배포 스크립트 - 변경 사항을 GitHub에 즉시 반영합니다.

---

## 사용법

```bash
/배포
```

## 실행 흐름

### 1️⃣ 변경된 파일 확인
```bash
git status
```
- 스테이징되지 않은 파일 목록 출력
- 수정/추가/삭제된 파일 확인

### 2️⃣ 커밋 메시지 자동 생성
변경된 파일을 분석하여 자동으로 커밋 메시지 생성:

| 변경 유형 | 커밋 메시지 |
|----------|-----------|
| HTML 파일 추가 | `Update dashboard: [파일명]` |
| CSS/JS 수정 | `Fix: [기능명]` |
| 문서 수정 | `Docs: [문서명]` |
| 데이터 추가 | `Add data: [데이터명]` |
| 버그 수정 | `Fix bug: [버그명]` |
| 기능 추가 | `Feature: [기능명]` |

**예시:**
```
Update dashboard: 회사대시보드.html
Fix: chart 색상 반영
Docs: CLAUDE.md 업데이트
```

### 3️⃣ GitHub에 푸시
```bash
git add .
git commit -m "[자동 생성된 메시지]"
git push origin main
```

- 모든 변경 파일 스테이징
- 커밋 생성
- main 브랜치로 푸시
- Vercel 자동 배포 트리거

### 4️⃣ 배포 완료 메시지
```
✅ 배포 완료!

📊 변경 사항:
  - 파일: [개수]개 변경
  - 커밋: [커밋 해시]
  - 브랜치: main

🌐 배포 URL:
  https://p-pro.vercel.app

⏱️ Vercel 반영 시간: 약 30초 ~ 1분
```

---

## 주의사항

⚠️ **배포 전 확인:**
- 로컬에서 테스트 완료
- 실수로 커밋할 파일 없는지 확인
- 중요한 파일 백업

✅ **자동 처리:**
- git add . (모든 변경사항)
- commit 메시지 자동 생성
- push to origin/main
- Vercel 자동 배포

## 자주 묻는 질문

**Q: Vercel에 반영되지 않았어요**
- A: 새로고침(Ctrl+F5)을 시도하거나 1-2분 기다린 후 재시도하세요.

**Q: 특정 파일만 커밋하고 싶어요**
- A: `/배포` 대신 `git add [파일명]` 사용

**Q: 커밋 메시지를 직접 작성하고 싶어요**
- A: `git commit -m "메시지"` 후 `git push`
